# AGENTS.md

## Localization

- Always use i18n for all user-facing text.
- Do not add hardcoded UI copy, alerts, button titles, labels, onboarding text, paywall text, status text, or error messages directly in source files.
- Route all visible strings through the project's localization system and add new keys when needed.

## Build Usage Rule

- Prefer the existing repo build and simulator commands or scripts before inventing new command sequences.
